# FinManager — Brainstorming Results

## Ключевые решения

### Dashboard
- Сводка по ВСЕМ ИП при входе: остатки по счетам, общий приход/расход за месяц, последние операции
- Summary cards сверху (glassmorphism карточки)
- Графики: timeline баланса + pie/bar по категориям
- Виджеты банковских карт (визуальное представление счетов)

### ДДС (ввод операций)
- **Модальное окно (wizard)** — пошаговый ввод как в боте, но красивее
- **Шаблоны (Templates)** — ключевая фича:
  - Пользователь один раз настраивает свои данные: ИП, счета, категории, статьи
  - Создаёт шаблоны для частых операций (например: "Закупка с Тиньк ИП Алексеев")
  - При вводе: выбрал шаблон → заполнил только сумму + номер заказа → готово
- **Все справочники редактируемые через UI:**
  - ИП (юр. лица)
  - Счета и карты (привязка к ИП)
  - Типы затрат (Закупка, Накладные, Маркетинг, Админ + пользовательские)
  - Статьи внутри типов (редактируемые)

### PDF-загрузка
- Drag & drop или кнопка "Загрузить"
- Выбор банка перед загрузкой (Сбер / Т-Банк / Депозит)
- **Превью-таблица** распарсенных данных перед сохранением
- Пользователь может проверить, исправить, удалить строки
- Кнопка "Подтвердить и сохранить"
- Batch upload (несколько PDF за раз) — в будущем
- Python-микросервис (FastAPI) для парсинга

### Уведомления
- В приложении: колокольчик в хедере, бейдж с числом
- Напоминания о загрузке выписок (2/12/22 числа)
- Уведомления о новых операциях от сотрудников
- Без email пока

### Мультиязычность
- Русский + English
- Переключатель в настройках профиля

### Роли и права
- Владелец: регистрация → настройка ИП/счетов/категорий → полный доступ
- Сотрудник: приглашение → гранулярные права (галочки):
  - ДДС ввод
  - PDF загрузка
  - Аналитика просмотр
  - Экспорт данных

### Дизайн
- Glassmorphism (полупрозрачные карточки, blur)
- Dark / Light theme (toggle)
- Adaptive: desktop + mobile (sidebar → bottom tabs)
- Accent: teal/бирюзовый
- i18n: ru/en

## Преимущества над ботами
1. Один интерфейс для всех ИП (вместо 3 отдельных ботов)
2. Шаблоны ДДС (вместо 6-8 шагов каждый раз)
3. Превью PDF перед сохранением (вместо "парсить и молиться")
4. Графики и аналитика в реальном времени
5. Настраиваемые справочники (вместо хардкода)
6. Нормальная БД (вместо Google Sheets с лимитами)
7. Система прав (вместо хардкода user_id)
